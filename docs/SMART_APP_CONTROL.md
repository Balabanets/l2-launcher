# Smart App Control (SAC) блокирует запуск игры

## Что это
Smart App Control — защита Windows 11, которая блокирует **неподписанные** программы
на этапе загрузки, опираясь на облако Microsoft. У L2-клиента `L2.exe` и его DLL
(`MinHook.x86.dll`, `AuthLogin746.dll` и др.) подписи нет → SAC их блокирует.

Признак: всплывает «**Smart App Control has blocked part of this app**».

## Чем SAC отличается от антивируса
- Это **НЕ** Defender. Исключения Defender (`Add-MpPreference`) на SAC не действуют.
- У SAC **нет белого списка** — ни игрок, ни программа не могут добавить файл в доверенные.
- Приложение **не может** выключить SAC само (Microsoft это запрещает; попытка только
  усилит блокировки). Отключается только вручную и **необратимо**.

## Кого касается
Только Windows 11 с **включённым** SAC — это редкость: он включается лишь на чистых
установках Win11 и часто сам уходит в выключенное состояние. На большинстве ПК его нет.

## Как выключить SAC (для игрока)
1. **Пуск** → введите `Smart App Control` (или «Управление интеллектуальными приложениями»)
   → откройте результат.
   Альтернатива: Безопасность Windows → «Управление приложениями и браузером» →
   «Параметры Smart App Control».
2. Переключите в **Off / Выключить**.
3. Подтвердите предупреждение.

⚠️ Необратимо: обратно SAC включается только переустановкой/сбросом Windows.

## Правильное (универсальное) решение — code signing
Чтобы игра работала и при включённом SAC, нужно подписать `L2.exe` и все неподписанные
DLL Authenticode-сертификатом, которому доверяет облако Microsoft (свежий сертификат
может набирать репутацию; EV-сертификат надёжнее). Бесплатный вариант — SignPath OSS
для публичных репозиториев. Это долго/сложно, но единственный путь без действий игрока.
